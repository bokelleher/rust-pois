Overview

POIS is an ESAM (Event Signaling and Management, SCTE 130-3) server that processes SCTE-35 signals and applies rule-based conditioning. It provides:

  • ESAM Signal Processing - Handle SignalProcessingEvent XML, return a SignalProcessingNotification
  • Rule-Based Conditioning - Match by command, segmentation type/UPID, signal ID glob, or UTC window; then pass-through, replace, delete, blackout, regionalize, shorten/extend/fill, slate, or tracking-only
  • Faithful SCTE-35 editing - In-place rewrite of UPID, break duration, and delivery-restriction flags (lengths + CRC recomputed), plus a built/literal payload builder
  • Decision metadata - A standard ESAM verb on the wire (replace/delete/noop) plus an optional <pois:Decision> element carrying the friendly action + params for a downstream packager
  • SESAME security (SCTE 130-9) - Optional HMAC auth, channel-scoped authorization, and AES-256-GCM payload encryption over the ESAM exchange (see below)
  • SCTE-35 tools - Build / decode / validate (base64, hex, or binary input) — see the Tools page
  • Multi-Channel + Event Logging + RESTful API - Independent rule sets per channel, full event history, and management endpoints

Authentication

All API endpoints (except ESAM) require Bearer token authentication:

Authorization: Bearer your-token-here

The ESAM endpoint (/esam) uses its own protection: by default it is unauthenticated (regular SCTE 130-3 ESAM), but per-channel SESAME (SCTE 130-9) can require a minimum security tier — HMAC-SHA256 auth (Tier 1), channel-scoped authorization (Tier 2), and AES-256-GCM payload encryption (Tier 3) — carried in HTTP headers with no ESAM XML schema change. Responses are signed (and, for Tier 3, encrypted) regardless, so a consumer can verify the POIS decision wasn't forged.

Common Use Cases

Here are practical examples of how to use POIS for real-world scenarios:

1
Convert Program End to Ad Break
Scenario: Your playout system sends program_end signals, but your ad insertion system expects splice_insert for ad breaks.

Solution: Create a rule that transforms program_end to splice_insert with a 60-second break duration:

Name:
Program End → 60s Ad Break
Action:
replace
Match:
{"anyOf":[{"scte35.command":"program_end"}]}
Parameters:
{"command":"splice_insert","duration_ticks":5400000,"pts_adjustment":0}

Result: Every program_end signal is automatically converted to a 60-second ad opportunity.

2
Filter Out Internal Test Signals
Scenario: Your testing team sends SCTE-35 signals with "test-" prefixed IDs that should not reach production ad servers.

Solution: Create a high-priority rule to delete any signals with test IDs:

Name:
Delete Test Signals
Priority:
1 (highest)
Action:
delete
Match:
{"anyOf":[{"acquisition_signal_id":"test-*"}]}

Result: All signals with IDs starting with "test-" are silently dropped before reaching downstream systems.

3
Adjust Ad Break Duration
Scenario: Your network requires all ad breaks to be exactly 90 seconds, but upstream signals vary between 60-120 seconds.

Solution: Create a rule that normalizes all splice_insert durations to 90 seconds:

Name:
Normalize to 90s Breaks
Action:
replace
Match:
{"anyOf":[{"scte35.command":"splice_insert"}]}
Parameters:
{"command":"splice_insert","duration_ticks":8100000}

Result: All splice_insert signals are rewritten to have exactly 90 seconds duration (8.1M ticks at 90kHz).

4
Add PTS Adjustment for Encoding Delay
Scenario: Your encoding pipeline introduces a 2-second delay, and SCTE-35 timing needs to be adjusted to compensate.

Solution: Create a rule that adds 180,000 ticks (2 seconds at 90kHz) to all signals:

Name:
Add 2s Encoding Delay
Action:
replace
Match:
{"anyOf":[{}]}
Parameters:
{"pts_adjustment":180000}

Result: All SCTE-35 signals have their timing adjusted by 2 seconds to synchronize with the encoding delay.

5
Route Different Channels to Different Ad Servers
Scenario: You have sports content on Channel A (using AdServer1) and news on Channel B (using AdServer2), each requiring different signal formats.

Solution: Create separate channels with custom rules:

Channel A:
sports-live
Rule:
Convert all to 30s breaks (AdServer1 format)
Parameters:
{"command":"splice_insert","duration_ticks":2700000}
Channel B:
news-24x7
Rule:
Convert all to program_start/program_end (AdServer2 format)
Parameters:
{"command":"program_start"}

Result: Each channel independently transforms signals according to its downstream ad server requirements.

# Sports channel uses sports-live rules POST /esam?channel=sports-live # News channel uses news-24x7 rules POST /esam?channel=news-24x7

ESAM Endpoint

POST /esam?channel={channel_name}

Process SCTE-35 signal through channel rules

Query Parameters

Parameter Type Required Description
channel string Yes Channel name to use for processing

Request Body

XML SignalProcessingEvent. The SCTE-35 payload is carried as base64 in <BinaryData signalType="SCTE35">; acquisitionPointIdentity typically names the channel.

<?xml version="1.0"?> <SignalProcessingEvent xmlns="urn:cablelabs:iptvservices:esam:xsd:signal:1"> <AcquiredSignal acquisitionSignalID="test-001" acquisitionPointIdentity="sportsfeed-east"> <UTCPoint utcPoint="2026-06-02T20:00:00Z"/> <BinaryData signalType="SCTE35">/DAhAAAAAAAAAP/wEAUAAAAAf+//AAAAAH4AAAAAAAA1FwHt</BinaryData> </AcquiredSignal> </SignalProcessingEvent>

Response

XML SignalProcessingNotification. The ResponseSignal action attribute is always a standard ESAM verb (replace, delete, or noop); the conditioned SCTE-35 is returned in <sig:BinaryData> (omitted for delete). When a rule matches, an optional <pois:Decision> element echoes the friendly action and its params for a downstream packager (ignored by stock encoders).

<?xml version="1.0"?> <SignalProcessingNotification xmlns="urn:cablelabs:iptvservices:esam:xsd:signal:1" xmlns:sig="urn:cablelabs:md:xsd:signaling:3.0" xmlns:core="urn:cablelabs:md:xsd:core:3.0" xmlns:common="urn:cablelabs:iptvservices:esam:xsd:common:1" xmlns:pois="urn:techex:pois:decision:1"> <common:StatusCode classCode="0"><core:Note>replaced signal</core:Note></common:StatusCode> <ResponseSignal action="replace" acquisitionSignalID="test-001" acquisitionPointIdentity="sportsfeed-east"> <sig:UTCPoint utcPoint="2026-06-02T20:00:00Z"/> <sig:BinaryData signalType="SCTE35">/DBLAAAAAAAAAP/wEAUAAAAB...</sig:BinaryData> <pois:Decision action="blackout">{"restrictions":{"no_regional_blackout":false}}</pois:Decision> </ResponseSignal> </SignalProcessingNotification>

Rule Actions

The action is a friendly label; on the ESAM wire it maps to a standard verb and the effect is carried in the conditioned SCTE-35 (plus <pois:Decision> metadata).

Action ESAM verb Effect
noopnoopPass the original signal through unchanged
replacereplaceReplace with a built / literal payload, or rewrite the incoming UPID
deletedeleteDrop the signal (no BinaryData emitted)
blackoutreplaceSet delivery-restriction flags on the incoming cue
regionalizereplaceRewrite the incoming UPID (regional/affiliate targeting)
shorten / extendreplaceSet the break duration (break_duration / segmentation_duration)
fillreplacePad the break to a target duration (filler is a packager hint)
slatenoopPass through + signal the MCR/packager to insert a slate
tracking-onlynoopPass through + report only (no ad decisioning)

Unknown actions fall back safely to noop. blackout/regionalize/shorten/extend/fill edit the incoming cue in place (lengths + CRC are recomputed); if there is no cue to condition the original is passed through.

Match Criteria (match_json)

Combine leaf conditions with anyOf (OR) and/or allOf (AND). An empty object {} is a deliberate catch-all. Supported leaf keys:

KeyMatch
acquisitionSignalIDGlob (* wildcards, e.g. blk-*, *AFE1*)
scte35.commandCommand name, case-insensitive (splice_insert, time_signal, …)
scte35.segmentation_type_idHex id, e.g. 0x34
scte35.segmentation_upidGlob against the decoded UPID
utcBetween{"start":"…Z","end":"…Z"} ISO-8601 UTC window
// ANY splice_insert OR a Program Start/End segmentation descriptor {"anyOf":[{"scte35.command":"splice_insert"},{"scte35.segmentation_type_id":"0x10"}]} // ALL of: signal id glob AND a prime-time window {"allOf":[{"acquisitionSignalID":"prod-*"},{"utcBetween":{"start":"2026-06-02T18:00:00Z","end":"2026-06-02T23:00:00Z"}}]} // Catch-all (matches every signal) {}

Action Parameters (params_json)

// replace — build a fresh avail (splice_insert ad-out), optional segmentation {"build":{"command":"splice_insert_out","duration_s":30, "segmentation_type_id":"0x34","upid_type":"0x0C","upid":"ABCD1234"}} // replace — build an immediate time_signal {"build":{"command":"time_signal_immediate"}} // replace — literal SCTE-35 payload (base64) {"scte35_b64":"/DAh..."} // replace / regionalize — rewrite the incoming UPID in place (type optional = keep) {"rewrite_upid":{"upid":"region-east-001","upid_type":"0x0C"}} // (replace) {"upid":"region-east-001","upid_type":"0x0C","region":"us-east"} // (regionalize) // blackout — set delivery-restriction flags {"restrictions":{"web_delivery_allowed":false,"no_regional_blackout":false, "archive_allowed":true,"device_restrictions":0}} // shorten / extend — set break duration (seconds); fill — pad to a target {"duration_s":15} {"to_duration_s":30,"filler":"house"} // slate — packager hint; noop / delete / tracking-only — no params {"asset":"slate-1","duration_s":10} {}

Code Examples

cURL Examples

Send ESAM Request

curl -X POST "http://localhost:3090/esam?channel=live-sports" \ -H "Content-Type: application/xml" \ -d '<?xml version="1.0"?> <SignalProcessingEvent xmlns="urn:cablelabs:iptvservices:esam:xsd:signal:1"> <AcquiredSignal acquisitionSignalID="test-001"> <UTCPoint utcPoint="2025-11-05T20:00:00Z"/> </AcquiredSignal> </SignalProcessingEvent>'

Create Channel

curl -X POST "http://localhost:3090/api/channels" \ -H "Authorization: Bearer dev-token" \ -H "Content-Type: application/json" \ -d '{"name":"live-sports","enabled":true}'

Create Rule

curl -X POST "http://localhost:3090/api/channels/1/rules" \ -H "Authorization: Bearer dev-token" \ -H "Content-Type: application/json" \ -d '{ "name": "Delete splice_insert", "priority": 10, "action": "delete", "match_json": "{\"anyOf\":[{\"scte35.command\":\"splice_insert\"}]}", "params_json": "{}" }'

Get Events

curl -H "Authorization: Bearer dev-token" \ "http://localhost:3090/api/events?channel=live-sports&limit=10"
