Overview

POIS is an ESAM (Event Signaling and Management) server that processes SCTE-35 signals and applies rule-based transformations. It provides:

  • ESAM XML Endpoint: Receives and processes SignalProcessingEvent messages
  • REST API: Manages channels, rules, and views event history
  • Rule Engine: Match conditions and transform signals
  • Event Logging: Full audit trail with raw XML storage

Base URL

http://localhost:8090 (development)
https://pois.techexlab.com:8090 (production)

Authentication

All API endpoints (except /health) require Bearer token authentication.

Authorization: Bearer dev-token

Default Token: dev-token (development only)

Set custom token using POIS_AUTH_TOKEN environment variable.

Common Use Cases

Real-world examples of POIS rule configurations for common broadcast scenarios.

1
Block All Upstream Ad Markers
Scenario: Upstream provider sends ad markers you want to ignore completely.
Name:
Block Upstream Ads
Priority:
0
Action:
delete
Match JSON:
{ "anyOf": [ {"scte35.segmentation_type_id": "0x30"}, {"scte35.segmentation_type_id": "0x32"} ] }
Params JSON:
{ } (empty - pass through unchanged)

Result: All provider (0x30) and distributor (0x32) ad markers are deleted before reaching downstream systems.

2
Convert Program Boundaries to Ad Breaks
Scenario: Network sends program end markers; you want to insert 60-second local ad breaks.
Name:
Program End → 60s Ad
Priority:
10
Action:
replace
Match JSON:
{ "allOf": [ {"scte35.command": "time_signal"}, {"scte35.segmentation_type_id": "0x11"} ] }
Params JSON:
{ "build": { "command": "splice_insert_out", "duration_s": 60 } }

Result: Every program end signal (0x11) is transformed into a 60-second splice_insert ad break.

3
Pass Through Premium Content Only
Scenario: Only process signals tagged with premium sports content identifiers.
Name:
ESPN Premium Only
Priority:
5
Action:
replace
Match JSON:
{ "allOf": [ {"scte35.command": "time_signal"}, {"scte35.upid": "*ESPN_PREMIUM*"} ] }
Params JSON:
{ } (empty - pass through unchanged)

Result: Only signals containing "ESPN_PREMIUM" in the UPID are passed through unchanged.

4
Tag Signals with Network ID
Scenario: Add your network identifier to all outgoing signals for tracking.
Name:
Add Network UPID
Priority:
20
Action:
replace
Match JSON:
{ "anyOf": [ {"scte35.command": "time_signal"} ] }
Params JSON:
{ "build": { "command": "time_signal", "upid_type": "TI", "upid_value": "YOUR-NETWORK-12345", "use_extracted_pts": true } }

Result: All time_signal commands get your network UPID injected while preserving the original PTS timing.

5
Variable Ad Break Durations
Scenario: Premium content gets longer ad breaks than standard content.

Strategy: Use multiple rules with different priorities - first match wins.

Rule 1: Premium Content (Higher Priority)

Priority:
5
Match:
{"scte35.upid": "*PREMIUM*"}
Params:
{"build": {"command": "splice_insert_out", "duration_s": 120}}

Rule 2: Standard Content (Lower Priority)

Priority:
10
Match:
{"anyOf": [{"scte35.command": "time_signal"}]}
Params:
{"build": {"command": "splice_insert_out", "duration_s": 60}}

Result: Premium content gets 120-second breaks, everything else gets 60-second breaks.

6
Delete Test Signals
Scenario: Block test signals from reaching production downstream systems.
Name:
Delete Test Signals
Priority:
0
Action:
delete
Match JSON:
{ "acquisition_signal_id": "test-*" }
Params JSON:
{ } (empty - pass through unchanged)

Result: Any signal with acquisition ID starting with "test-" is deleted.

ESAM Endpoint

POST /esam?channel={channel_name}

Main ESAM endpoint that receives SignalProcessingEvent XML and returns SignalProcessingNotification XML based on matched rules.

Query Parameters

Parameter Type Required Description
channel string Yes Channel name to process signal for

Request Body

<?xml version="1.0"?> <esam:SignalProcessingEvent xmlns:esam="urn:cablelabs:iptvservices:esam:xsd:signal:1" xmlns:md="urn:cablelabs:md:xsd:signaling:3.0"> <esam:AcquiredSignal acquisitionSignalID="test-001" acquisitionPointIdentity="station-01" acquisitionTime="2025-11-05T20:00:00Z"> <md:UTCPoint utcPoint="2025-11-05T20:00:00Z"/> <md:BinaryData signalType="SCTE35">/DAlAAAAAAAAAP/wFAVAAAsxf+//+pTQbf4ApO6oAGkAAAAAUIWI+A==</md:BinaryData> </esam:AcquiredSignal> </esam:SignalProcessingEvent>

Response (200 OK)

<SignalProcessingNotification xmlns="urn:cablelabs:iptvservices:esam:xsd:signal:1"> <StatusCode classCode="0"> <Note>success</Note> </StatusCode> <ResponseSignal action="noop" acquisitionSignalID="test-001"> <UTCPoint utcPoint="2025-11-05T20:00:00Z"/> <BinaryData signalType="SCTE35">/DAlAAAAAAAAAP/wFAVAAAsxf+//+pTQbf4ApO6oAGkAAAAAUIWI+A==</BinaryData> </ResponseSignal> </SignalProcessingNotification>

Channels API

GET /api/channels

List all configured channels

Response

[ { "id": 1, "name": "live-sports", "enabled": true, "created_at": "2025-11-05T10:00:00Z", "rules": [...] } ]
POST /api/channels

Create a new channel

Request Body

{ "name": "live-sports", "enabled": true }
PUT /api/channels/{channel_name}

Update channel (enable/disable)

Request Body

{ "enabled": false }
DELETE /api/channels/{channel_name}

Delete a channel and all its rules

Rules API

POST /api/channels/{channel_name}/rules

Create a new rule for a channel

Request Body

{ "name": "Delete splice_insert", "priority": 10, "enabled": true, "action": "delete", "match_json": "{\"anyOf\":[{\"scte35.command\":\"splice_insert\"}]}", "params_json": "{}" }

Match JSON Examples

// Match any splice_insert command {"anyOf":[{"scte35.command":"splice_insert"}]} // Match program start (type 0x10) {"allOf":[{"scte35.command":"time_signal"},{"scte35.segmentation_type_id":"0x10"}]} // Match ESPN content {"scte35.upid":"*ESPN*"} // Complex: Match provider ads for premium content { "allOf": [ {"scte35.segmentation_type_id":"0x30"}, {"scte35.upid":"*PREMIUM*"} ] }

Params JSON Examples

// Pass through unchanged {} // Build 60s splice insert {"build":{"command":"splice_insert_out","duration_s":60}} // Use extracted PTS with 5s offset {"build":{"command":"splice_insert_in_with_pts","use_extracted_pts":true,"pts_offset_s":5}} // Replace with pre-built SCTE-35 {"scte35_b64":"/DAlAAAAAAAAAP/wFAVAAAsxf+//+pTQbf4ApO6oAGkAAAAAUIWI+A=="}
PUT /api/channels/{channel_name}/rules/{rule_id}

Update an existing rule

DELETE /api/channels/{channel_name}/rules/{rule_id}

Delete a rule

Events API

GET /api/events

List processed events with optional filtering

Query Parameters

Parameter Type Default Description
limit integer 100 Number of events (max 1000)
offset integer 0 Skip this many events
channel string - Filter by channel name
action string - Filter by action (noop, delete, replace)
since string - Time period (1h, 24h, 7d, 30d)
GET /api/events/{event_id}

Get full event details including raw XML

Response

{ "id": 123, "timestamp": "2025-11-05T20:00:00Z", "channel_name": "live-sports", "acquisition_signal_id": "test-001", "action": "replace", "matched_rule_name": "Program End → 60s Ad", "raw_esam_request": "<SignalProcessingEvent>...</SignalProcessingEvent>", "raw_esam_response": "<SignalProcessingNotification>...</SignalProcessingNotification>" }
GET /api/events/stats

Get aggregate event statistics

Response

{ "total_events": 1234, "events_last_24h": 567, "avg_processing_time_ms": 1.5, "by_action": { "noop": 800, "delete": 200, "replace": 234 } }

Code Examples

cURL Examples

Send ESAM Request

curl -X POST "http://localhost:8090/esam?channel=live-sports" \ -H "Content-Type: application/xml" \ -d '<?xml version="1.0"?> <SignalProcessingEvent xmlns="urn:cablelabs:iptvservices:esam:xsd:signal:1"> <AcquiredSignal acquisitionSignalID="test-001"> <UTCPoint utcPoint="2025-11-05T20:00:00Z"/> </AcquiredSignal> </SignalProcessingEvent>'

Create Channel

curl -X POST "http://localhost:8090/api/channels" \ -H "Authorization: Bearer dev-token" \ -H "Content-Type: application/json" \ -d '{"name":"live-sports","enabled":true}'

Create Rule

curl -X POST "http://localhost:8090/api/channels/live-sports/rules" \ -H "Authorization: Bearer dev-token" \ -H "Content-Type: application/json" \ -d '{ "name": "Delete splice_insert", "priority": 10, "action": "delete", "match_json": "{\"anyOf\":[{\"scte35.command\":\"splice_insert\"}]}", "params_json": "{}" }'

Get Events

curl -H "Authorization: Bearer dev-token" \ "http://localhost:8090/api/events?channel=live-sports&limit=10"
